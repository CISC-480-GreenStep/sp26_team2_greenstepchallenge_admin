# Documentation

Project documentation for the GreenStep Sustainability Challenge Admin Console.

## Start here

- **[`ARCHITECTURE.md`](./ARCHITECTURE.md)** — annotated source tree, "where do I
  change X?" recipes, and "how do I add Y?" playbooks. Read this first when
  onboarding or before opening a refactor PR.

## Contents

- `ARCHITECTURE.md` — contributor's map of the codebase (source of truth for
  file layout and navigation).
- `guides/` — focused, task-scoped onboarding guides for individual issues
  or areas of the app. Read the matching guide before starting on an issue.
  - [`guides/theme-design-system.md`](./guides/theme-design-system.md) — the
    slice of the codebase that owns theming, per-challenge colors, the
    mobile preview, and header images
    ([Issue #20](https://github.com/CISC-480-GreenStep/sp26_team2_greenstepchallenge_admin/issues/20)).
- `backups/` — backup copies of meeting documents and project files
  (pre-merge snapshots, etc.).

## Other root-level docs

- [`../README.md`](../README.md) — project overview, setup, version history,
  feature catalog, deployment notes.
- [`../CODING_GUIDELINES.md`](../CODING_GUIDELINES.md) — naming, imports,
  comments, file-size policy, code-review checklist.
